How scoring works

Every number is computed in plain JavaScript, in this file. The LLM never touches a score.

score = min(99, round( Σ weight × 0.5^(age_days / 21) × quality_prior + x_validation ))

Signal weights

Recency decay

Each signal decays with a 21-day half-life: full weight the day it fires, half after three weeks, a quarter after six. A person is interesting because of what happened recently, so old signals fade instead of accumulating forever.

Founder-quality prior (multiplier)

X validation (optional adjustment)

Runs only when you click Validate on X: cadence min(4, posts/week) + audience min(4, log10(followers)) + engagement (2 if median likes ≥ 10, 1 if ≥ 3), cap +10. The receipt labels the line sample or live; sample numbers exist only for the fictional candidates.

Scores cap at 99. "Likely" means 60 or above. Live Hacker News candidates run through the identical engine.

Code owns the facts, the LLM owns the language. Statuses, scores and rankings are deterministic; the model only writes prose, and every LLM feature has a deterministic fallback.
